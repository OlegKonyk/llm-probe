🧠 TESTING LLM APIs - KEY TALKING POINTS

The Challenge:

  • Same prompt ≠ same output (non-deterministic)
  • Hallucinations (plausible but wrong)
  • Latency variability
  • Bias detection

Your Approach:

❌ DON'T: assert response == "exact text"

✓ DO:

  • Semantic similarity (threshold ≥ 0.85)
  • Structural validation (JSON schema)
  • Statistical variance checks
  • LLM-as-a-judge patterns
  • Golden dataset regression testing
  • Hallucination detection

Testing Layers:

  1. Contract tests (API schema validation)
  2. Functional (happy path + edge cases)
  3. Performance (latency, throughput, cost-per-call)
  4. Chaos/Resiliency (timeouts, retries, degradation)
  5. Security (prompt injection, PII leakage)
